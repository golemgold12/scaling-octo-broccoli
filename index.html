Web Dev Concepts

Definition of Terms

Website: a set of webpages that are located under a singular domain name, usually interlinked with each other. They are named this due to the World Wide Web, or WWW.
Sites can be static(unchanging), or dynamic.



Web Application: A service that is offered and runs in your web browser. A example could be Youtube or online shopping.



Web Browser: A application used to search the web, retreving data from web servers to present it in the form of web pages.
Browsers send HTTP requests to servers via URLs, so the server can display the page.



Web Server: A server that accepts HTTP or HTTPS requests to deliver web content to a client.
Web servers deliver content via HTTP responses.



HTTP Request: A message sent to a web server in a specific format- in order to get a response from the web server. Whenever you visit a website, a request is sent.
All HTTP requests have 4 parts: Request Line, Header, a blank line, and a body.



HTTP Response: Whenever the server recieves a HTTP request, it sends data back in the form of a respnse. Web pages are generated via HTTP responses to HTTP requests.
HTTP responses have 4 parts: Status Line, response header, blank line, and a optional body,

Frontend Design

Frontend design focuses on the graphical design and user interface of a website. User interfaces and the overall aestetic of a website all pertain to frontend design. This is a concept that also combines a bit of human aspects- frontend devs usually have to design a interface that is easy to naviagate for the average user.


Five E's of Usability:

Effectiveness
The purpose of something
Efficency
How versatile and easy to use it is
Engagement
How it interacts with the client/user
Easy to learn
Is it easy to understand?
Error tolerance
How accepting is it of errors? How prone is it to bugs?

Page Layout Tags: Page tags like "header" and "section" are tags that can be used to organize a website's content, therfore making it easier to use. Without them, everything would look jumbled.



Tag Description
Header A tag that usually is reserved for the title of something. Appeaars in bigger text.
Paragraph A tag for long stretches of text- or just paragraphs.
Nav A tag for a list of hyperlinks. Easy to navigate.
Footer A tag usually reserved for the very bottom parts of a page.
Table A tag that automatically creates a chart-like table. Great for... tables.
Iframe Embeded frame. Used for a page within a page.
